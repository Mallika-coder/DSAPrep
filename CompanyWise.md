## OA (Online Assessment) Common Patterns

These types appear across most company OAs:

| Category | Typical Problems | Key Pattern |
|----------|-----------------|-------------|
| Array Manipulation | Subarray sum, max subarray, rotate | Prefix Sum, Kadane's |
| String Processing | Anagram check, palindrome, matching | HashMap, Two Pointers |
| Graph/Grid | Number of islands, shortest path, connected components | BFS/DFS |
| Greedy | Minimum cost, activity selection, scheduling | Sort + Greedy rule |
| Binary Search | Allocate resources, minimize maximum | BS on Answer |
| Tree | LCA, diameter, traversals | DFS |
| DP | Longest subsequence, coin change, knapsack | Standard DP patterns |
| Math | Modular arithmetic, combinatorics | Number theory |

### OA Tips 🔥
```
1. Read ALL problems first — start with the easiest
2. Get brute force working FIRST for partial marks
3. Edge cases matter more in OAs — test with minimum inputs
4. Time management: Don't spend >20 min on one problem
5. Common OA traps:
   - Integer overflow (use long/long long)
   - Off-by-one in binary search
   - Empty input / single element
   - Forgetting to sort before binary search
   - Not handling disconnected graph components
```

## 📊 Priority Order for Preparation

Based on your profile (MNNIT, Amazon intern, targeting Google-level):

```
HIGH PRIORITY (do these first):
├── Binary Search on Answer (5+ problems)
├── Sliding Window (all variants)
├── Graph BFS/DFS (grids + adjacency)
├── DP (Knapsack, LIS, LCS patterns)
├── Monotonic Stack
└── Two Pointers

MEDIUM PRIORITY:
├── Heaps / Priority Queue
├── Backtracking (with duplicate handling)
├── Greedy (with proofs)
├── Trees (all traversals + paths)
└── Union-Find

PRACTICE REGULARLY:
├── Codeforces Div 2 A/B/C (2 per day)
├── Edge case thinking
└── Speed coding (time yourself)
```
