## 🧠 How to Solve Each Problem (Step by Step)

```
Step 1: READ the problem — understand input/output/constraints (2 min)
Step 2: IDENTIFY the pattern from the table tag (1 min)
Step 3: THINK for 15 min max — dry run examples, think edge cases
Step 4: If stuck after 15 min → read the 💡 hint in the sheet → try 10 more min
Step 5: If still stuck → look at the APPROACH (not code!) → code it yourself
Step 6: After solving → mark [x], write 1-line insight
Step 7: If it took >30 min or you couldn't solve → star it ⭐ for revisit in 2 days
```

### The 15-Minute Rule
- Can't think of approach in 15 min → read pattern tag → try again 10 min → look at approach (NOT code) → code yourself
- NEVER copy-paste code. Even if you read the solution, close it and type from understanding.

## 💡 Pro Tips

1. **Track patterns, not problem count.** Solving 50 problems badly < solving 20 problems with deep understanding.

2. **Always optimize.** After brute force works, ask: "Can I do better? What data structure helps?"

3. **Think in templates.** Each pattern has a code template (see PatternCheatSheet.md). Memorize them:
   - Sliding Window template
   - Binary Search on Answer template
   - Backtracking template
   - Monotonic Stack template
   - DFS on Tree template
   - BFS template

4. **Verbalize your approach.** Before coding, explain your approach in 3 sentences. If you can't explain it simply, you don't understand it yet.

5. **Speed builds with repetition.** First time a pattern takes 40 min. Third time it takes 15 min. Tenth time it takes 5 min.

6. **For Google-style prep:** Focus on Binary Search on Answer, Greedy with proofs, Graph BFS/DFS variations, DP optimizations.

7. **For Amazon-style prep:** Focus on HashMap tricks, Monotonic Stack, Priority Queue, BFS on grids.
