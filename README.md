## 🧠 How to Solve Any Problem (Framework)

```
1. READ the problem carefully — identify input/output/constraints
2. IDENTIFY the pattern — what technique does this fall under?
3. THINK about edge cases — empty, single element, all same, overflow, boundaries
4. BRUTE FORCE first — get a working solution
5. OPTIMIZE — can you reduce time/space? What data structure helps?
6. CODE it cleanly — variable names, edge case handling
7. DRY RUN on examples + edge cases
8. ANALYZE complexity — Time & Space
```

## 🎯 Key Tips

- **Don't just solve — TAG every problem** with its pattern
- **Edge cases mantra:** empty, single, all same, overflow, k=0, k=1, negative numbers
- **After solving:** "What was the ONE insight that cracked this?" — write it
- **For Google-style:** Binary Search on Answer, Greedy with proofs, Graph BFS/DFS variations, DP optimizations
- **For Amazon-style:** HashMap tricks, Monotonic Stack, Priority Queue, System Design thinking
- **Speed:** If you can't think of approach in 15 min, read the hint/pattern, then code yourself
